2.1 Making some charts outside of the school
As in the last lesson, throwing some code at you, the reader, and then explaining is a good way to break the ice. So, ice breaker:
<div id='chart'></div> var w = 600; var h = 100; var root = d3.select('#chart').append('svg') .attr({ 'width': w, 'height': h, }) .style('border', '1px solid black'); var evenNumbers = [2, 4, 6, 8, 10]; var barHeight = h / evenNumbers.length; root.selectAll('rect.number') .data(evenNumbers).enter() .append('rect') .attr({ 'class': 'number', 'x': 0, 'y': function(d, i) { return i * barHeight; }, 'width': function(d) { return 60 * d; }, 'height': barHeight, 'fill': '#dff', 'stroke': '#666', });
Our first two lines are simple: we set a variable named w and one named h to some numbers. Why? Who cares, if it doesn't make sense, keep reading. Doing so, we get: var root = d3.select('#chart').append('svg') .attr({ 'width': w, 'height': h, }) .style('border', '1px solid black'); New syntax! Join the club. D3 is under constant development and new features get added with some frequency. Your author tries to keep up, but can't always be bothered. The two variables from before should now be more obvious in their meaning - they are the width and height of whatever chart we are making. Why not give them a longer name? A better question: why name them at all? Aside from it being good style to do so, we also still have use for them.
